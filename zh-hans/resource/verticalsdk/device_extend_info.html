1.1. 拓展设备功能点介绍

目的由于即使是同一品类的设备，其功能点也会由于具体的设备差异，导致功能点不能完全一致。在以提供的设备控制能力上，开放一些功能接口，方便功能拓展

1.1.1. BaseDevice 功能介绍-品类扩展

能力介绍

该类是所有标准设备公用能力实现的基类，具体方法功能介绍参考：设备公共能力方法说明 .md

可以继承该类，实现对其它品类功能控制的能力

使用示例


public class OtherNewDevice extends BaseDevice {
    public OtherNewDevice(String deviceID) {
        super(deviceID);
    }
    public void  getSwitch(){}
    public void  getLight(){}
    public void  getOtherAbility(){}
}

public static void main(String[] args) {
    OtherNewDevice device = new OtherNewDevice("123");
    List<String> supportStatusList = device.getSupportStatusList();
    List<String> supportFunctionList = device.getSupportFunctionList();
    device.getLight();
}

1.2. ProductStandardConfig.StatusSchemaBean 介绍

成员类型 说明
HashMap relationDpIdMaps 设备关联的dp点和dpCode
String dpCode 设备的dpCode，上层不使用（使用的是standardCode），不同设备同一功能可能定义的dpCode不同，但standardCode是一致的 。
String standardType 数据的类型 如 Boolean、 Integer、 Enum、 String等
String strategyCode 定义的解析策略 如 default enum
String strategyValue 策略值，根据strategyCode类型解析出里面的标准指令standardCode
String valueRang 定义了某些数据的范围 如Integer类型的 {"min":0,"max":1000,"scale":0,"step":1}， Enum类型的{"range":["sleep","low","mid","high","auto"]}

获取示例

DeviceBean deviceBean = TuyaHomeSdk.getDataInstance().getDeviceBean(mDevId);
ProductStandardConfig productStandardConfig = TuyaHomeSdk.getDataInstance().getStandardProductConfig(deviceBean.productId);
// 示例 获取第一个数据 StatusSchemaBean
ProductStandardConfig.StatusSchemaBean statusSchemaBean = productStandardConfig.statusSchemaList.get(0);

1.3.5. 解析 strategyType是Enum类型的valueRange数据

protected List<String> getRangeArray(String StandardCode)

作用：解析类似 {"range":["sleep","low","mid","high","auto"]}类型的数据 转换成数组，定义了数据的状态

参数说明

返回值 说明 参数 说明
List<String> 返回数据取值的集合 String StandardCode 标准指令

示例代码

   public class MyDeviceParser extends DeviceParser {
        public MyDeviceParser(String deviceID) {
            super(deviceID);
        }

        public String getCurSpeed() {
            loadDpMap();
            String speed = getString("Speed_Code");
            return speed;
        } 

        public List<String> getCurSupportSpeeds() {
            List<String> speed_code = getRangeArray("Speed_Code");
            return speed_code;
        }
    }

1.3.6. 解析 strategyType是Interger类型的valueRange数据

public ValueRangeBean getValueSchema(String dpCode)

作用：解析类似 {"unit":"%","min":0,"max":100,"scale":0,"step":1} 类型的数据，转换成Bean，定义了Interger 类型数据范围

返回值 说明
String unit 单位
int min 数据最小值（需要使用scale缩放再使用）
int max 数据最大值（需要使用scale缩放再使用）
int scale 缩放因子
int step 步长 针对[min,max] 间的变化范围的

说明： 通过 DeviceParser 里面的方法获取值都是经过scale计算后的值 无需再处理

{"unit":"°C","min":0,"max":400,"scale":1,"step":1} 表示的含义为 单位是摄氏温度 ，最小值0 ，最大值40 ，温度变化最小值是 0.1

示例代码

// 计算方法
double del = Math.pow(10, bean.scale);
double min_scale = bean.min / del;
double max_scale = bean.max / del;
double temp_scale = temp / del;

// 数据上报时则需要转换成valueRange里面的范围数据
double upTemp =  temp_scale * del;
publish(upTemp);
