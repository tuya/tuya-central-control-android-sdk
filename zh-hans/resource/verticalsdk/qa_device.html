1. 常见问题Q&A和一些注意事项

1.1. 常见问题

Q：当监听到当前中控在手机上被移除后，重新调用扫码页面登录，子设备全部离线了？

A：当监听到中控被移除后，正确的方式通过重启再次进入扫码。原因是重新进入扫码页，已经初始化后网关（单例）创建的线程还在导致。

Q：部分敏感类型的传感类设备在 ''onDpUpdate''中，多次回调报警类dp信息？

A：后续会迭代更新掉，目前的建议做法是 DeviceBean中有个成员变量dpsTime Map dpsTime，记录了dp下发的时间（毫秒值）

        DeviceBean deviceBean = TuyaHomeSdk.getDataInstance().getDeviceBean("mDevID");
        long beforeDpIdTime = 0l;//这里先记录上一次的时间
        Map<String, Long> dpsTime = deviceBean.getDpsTime();
        Long dpIdTime = dpsTime.get("dpId");//警报功能对应的dpId  数字值
        if (beforeDpIdTime == dpIdTime) {
            // 说明存在重复，可以过滤掉这个警报信息，只更新其他信息
        }

Q：如何刷新列表或者是类似的操作？

A：通过 TuyaHomeSdk.newHomeInstance(homeId).getHomeDetail方法

     //会触发刷新的操作
     TuyaHomeSdk.newHomeInstance(homeId).getHomeDetail(new ITuyaHomeResultCallback() {
            @Override
            public void onSuccess(HomeBean bean) {
                List<DeviceBean> deviceList = bean.getDeviceList();
            }

            @Override
            public void onError(String errorCode, String errorMsg) {

            }
        });

Q：如何SDK的打开日志，查看一些关键日志？

//开启SDK 日志
TuyaHomeSdk.setDebugMode(true);

上报关键TAG
AbsTuyaDevice 设备发送出去的dp点日志
示例： AbsTuyaDevice dps: {"21":"white","22":710}

消息下发是mqtt系列消息日志关键TAG  
mqtt protocol  

客户端请求TAG
OKHttpBusinessRequest

Q：提供的品类API 中没有我设备的功能，为什么或者怎么办？

A：垂直SDK(即verticalcategory的依赖) 包含了主要了设备的主要功能，但是不同的设备可能都会有一些自己功能拓展并且符合涂鸦的标准指指令体系，目前暂时不能面面具到，后面会慢慢适配增加。

那么现阶段怎么办，这个功能不能用了？

针对这种情况，垂直sdk 部分把标准指令解析的部分开放出来，用户可以根据对应的规则自己解析数据，拓展设备功能和品类。详见 拓展设备功能点介绍 部分内容

注意： 前提是符合涂鸦标准指令体系的功能点

1.2. ⚠️注意事项：

1.tuyaIotBindToken 和 tuyaIotStart 在一个应用的生命周期内只能触发一次

TuyaIotSdk.getInstance().tuyaIotBindToken(token);
TuyaIotSdk.getInstance().tuyaIotStart(context,config);

当启动时就崩溃 出现了Fatal Single 相关的 并且日志中onStatusChanged 回调了多次(默认只有一次回调)，那么可以确定网关相关被初始化了多次导致的。

2.TuyaHomeSdk中 newXXInstacne 方法创建的对象都是重新New出单一个新对象 不是单例，使用时要注意避免创建多个对象
